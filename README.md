# 智慧生產物料預警系統 (Integrated Inventory Intelligence)

這是一個專為 **生管 (PC)**、**物管 (MC)** 與 **訂單管理 (OM)** 職位設計的技術展示專案。本專案演示了如何利用 **Python** 的強大邏輯運算與 **Chart.js** 的視覺化能力，填補傳統 ERP 系統在「動態決策」上的空缺。

# 核心價值：從「紀錄數據」到「預測風險」
大多數公司擁有 SAP 或 ERP 系統，但數據往往停留在靜態記錄。本專案實現了：
1. **數據膠水 (Data Glue)**：利用 Python 整合異質系統數據。
2. **自動化運算**：實作 ATP (Available To Promise) 邏輯，自動識別未來斷料點。
3. **情境模擬 (What-If)**：讓管理者透過網頁即時模擬需求波動對產線的衝擊。

---

# 技術架構 (System Architecture)

### 1. 後端數據處理 - [data_processor.py](./data_processor.py)
* **語言**：Python
* **核心功能**：
    * 模擬從 ERP 匯出的原始 JSON/CSV 大數據。
    * 執行物料需求計劃 (MRP) 運算。
    * 計算「預計可用量 (Projected On-Hand)」並標記風險狀態。
    * *面試說明：在實際環境中，此腳本可透過 Pandas 與 SQL 直接串接生產資料庫。*

### 2. 前端決策畫板 - [index.html](./index.html)
* **技術**：Chart.js, HTML5, CSS3
* **核心功能**：
    * **紅區管理 (Visual Management)**：當庫存水位低於 0 時，自動渲染紅色區塊，落實「例外管理」。
    * **即時交互**：滑桿微調功能，模擬訂單增減對供應鏈的壓力。
    * **Demo 連結**：[點擊觀看即時儀表板](https://chen-yu86.github.io/ERP-SAP/)

# 應用場景範例
* **生管會議**：不再對著 Excel 數字吵架，而是透過圖表直接討論「5/05 的斷料區如何透過調單或催料解決」。
* **採購預警**：根據 Python 計算出的缺料日，自動推算最晚下單日 (LOD)，避免緊急空運成本。
